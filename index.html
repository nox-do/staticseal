# self-encrypted

Seal and publish.

Wrap HTML onepagers, PDFs, and images in a static HTML shell that keeps the content out of plain crawler view. No backend, accounts, or extra tooling.

Add a passphrase for confidentiality, or leave it blank for a lightweight publishing veil against crawlers and link-preview bots. Either way, this is a static visibility layer, not access control.

No file selected

Leave both fields blank to seal for publishing without a password. This hides cleartext from crawlers, but anyone with the file can open it.

Recommended balances unlock speed with resistance to offline password guessing.
